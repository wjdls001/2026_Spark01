## 1. 프로젝트 개요

### 프로젝트명

**SPARK**

### 한 문장 서비스 정의

**10분 거리 이웃과의 부담 없는 연결로 운동의 불씨를 당기는, 하이퍼로컬 운동 행동 유도 플랫폼**

### 서비스가 아닌 것

SPARK는 단순 운동 기록 앱이 아니며, 친목 중심의 동호회 서비스도 아니다.  
핵심은 사용자가 운동을 “잘 기록”하게 만드는 것이 아니라, 실제로 운동을 “시작”하고 “지속”할 수 있게 만드는 것이다.

### 해결하려는 문제

2030 1인 가구 및 혼자 운동하는 사용자는 다음 이유로 운동을 지속하기 어렵다.

1. 운동을 시작하기까지의 심리적 부담이 크다.
2. 함께 운동할 사람이 없어 행동으로 이어지기 어렵다.
3. 정기 동호회는 부담스럽지만, 가볍게 운동할 사람은 필요하다.
4. 가까운 거리에서 지금 참여 가능한 운동 모임을 찾기 어렵다.

### 제공 가치

사용자는 거창한 준비 없이, 지금 내 주변 이웃과 빠르게 연결되어 운동을 시작할 수 있다.  
SPARK는 운동 기록보다 **운동 행동의 계기**를 제공하는 데 집중한다.

### 슬로건

**혼자서는 힘든 운동, 이웃과 함께 운동을 시작할 불씨를 키워볼까요?**

### North Star Metric

**주간 성사된 운동 스파크 수(Weekly Spark Counts)**

가입자 수나 게시글 수가 아니라, 실제 사용자가 이웃과 만나 운동을 시작한 행동의 총량을 핵심 지표로 본다.

## 2. 핵심 제품 원칙

### 2.1 운동 시작 우선

홈 화면과 주요 진입점에서 사용자가 가장 빠르게 운동을 시작할 수 있어야 한다.  
`운동 시작하기` CTA는 홈 화면 상단 또는 가장 눈에 띄는 위치에 배치한다.

### 2.2 하이퍼로컬 연결

사용자는 멀리 있는 사람이 아니라, 도보 또는 짧은 이동 거리 내의 이웃과 연결되어야 한다.  
위치 기반 탐색, 지도 보기, 리스트 보기, 거리 기반 필터를 우선 고려한다.

### 2.3 부담 없는 번개형 운동

정기 모임, 동호회, 깊은 친목보다 즉시성 있는 운동 연결에 집중한다.  
사용자는 가볍게 모임을 만들고, 신청하고, 승인받고, 운동을 완료할 수 있어야 한다.

### 2.4 신뢰와 안전

오프라인 만남이 발생하는 서비스이므로 신뢰도, 운동 레벨, 참여 이력, 신고, 후기, 취소 패널티를 고려한다.

### 2.5 기록은 행동 이후의 동기부여

운동 기록은 서비스의 중심 목적이 아니라, 운동 완료 이후 성취감과 재방문을 유도하는 보조 장치다.

## 3. Figma Page 1 기준 제품 흐름 보강

Figma Page 1 와이어프레임 분석 결과, SPARK의 제품 경험은 다음 순서로 이해한다.

```txt
소셜 로그인
→ 약관 동의
→ 전화번호 인증
→ 사용자 조사 7단계
→ 운동 레벨 결과
→ 홈
→ 번개 지도 탐색 또는 운동 시작
→ 운동 세션
→ 운동 기록/후기
→ 마이페이지에서 성장 확인
```

### 3.1 핵심 경험 재정의

SPARK의 핵심은 사용자가 홈에서 오래 탐색하게 만드는 것이 아니라, **지금 운동을 시작할 수 있는 가장 가까운 경로를 제공하는 것**이다. 따라서 번개 모임은 리스트보다 지도를 우선하고, 운동 시작은 복잡한 설정 페이지보다 종목 선택과 목표 입력을 한 화면에서 끝내는 구조를 우선한다.

### 3.2 온보딩의 역할

온보딩은 단순 프로필 입력이 아니라, 사용자에게 맞는 운동 레벨과 추천 흐름을 만들기 위한 사용자 조사다. 7단계 조사는 운동 목표, 성향, 선호/주 운동, 빈도, 평균 시간을 수집해 홈, 번개 추천, 운동 루틴 추천에 사용한다.

2026-06-18 확정 와이어프레임에서는 사용자가 조사 맥락을 잃지 않도록 모든 단계에 현재 단계, 퍼센트, 진행 바를 표시한다. 닉네임 중복 확인과 성별 추천 정확도 안내는 팝업으로 즉시 피드백하며, 운동 성향과 운동 종목에는 직접 입력을 제공한다. 화면 구조는 와이어프레임을 따르고 시각 표현은 SPARK 디자인 시스템의 Lime/Purple/Soft Surface 토큰을 적용한다. 완료 후에는 조사 결과를 운동 레벨·운동 취향·운동 빈도로 압축해 보여주고 바로 SPARK를 시작하게 한다.

### 3.3 기록의 위치

운동 기록은 서비스 진입 목적이 아니라 운동 이후의 성취 피드백이다. 운동 완료 후 기록, 레벨, XP, 후기, 통계가 마이페이지에서 재방문 동기로 작동해야 한다.

2026-06-18 홈/마이페이지 프레임 반영에서는 홈의 운동 시작 CTA를 성장 요약 직후에 두고 루틴·추천 번개·오늘 현황·최근 기록이 이어지게 한다. 마이페이지는 프로필 카드 자체를 편집 진입점으로 사용한다. 와이어프레임의 시각 스타일은 복제하지 않고 SPARK 디자인 시스템으로 해석한다.

운동 관리에서는 기록을 단순 목록으로만 보여주지 않고 빠른 재기록, 기간별 변화, 목표 달성 피드백의 세 관점으로 나눈다. 날짜 이동이 가능한 달력과 차트, 운동 종류별 상세 기록, 번개 동료 정보가 운동 이후의 성취감과 재방문 동기를 만든다.

기간별 비교에서는 거리와 시간뿐 아니라 칼로리와 페이스를 같은 위계로 제공한다. 연간 화면은 달력보다 월별 운동 횟수를 먼저 보여주어 장기적인 지속성을 빠르게 인지하게 한다. 지도 기반 상세 정보는 콘텐츠를 가리지 않도록 하단 슬라이드 카드로 열고 닫는다.
